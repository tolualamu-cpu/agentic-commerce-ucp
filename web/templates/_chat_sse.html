{# SSE wiring for the chat page. Lives in chat.html only — not on home.html or other pages. Appends bubbles to #chat-log and handles the dedup edge case where the page arrives from Explore with a server-rendered user bubble that the queued "user" event would otherwise duplicate. Phase 8f: this script no longer opens its own EventSource. The single per-page connection is owned by _toast.html, which re-broadcasts each non-cart_update frame as an `ac:sse` CustomEvent on the document. We subscribe to that DOM event. Reason: two EventSources to /chat/stream would split events across two queue consumers (only one would get each event). Scroll behaviour (Phase 8c): - Sticky to bottom: only auto-scrolls when the user is already within NEAR_BOTTOM_PX of the bottom. If they've scrolled up to re-read, streamed text deltas leave them alone. - Coalesced via requestAnimationFrame: at most one scroll per frame regardless of how many SSE deltas land. - Force-scroll on user submit: when the user adds their OWN bubble (just hit Send), bypass the near-bottom check. #}
